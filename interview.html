Topic 1: Git Basics

fundamentals (Google, Amazon, Microsoft)

10 Questions
Core

What is Git?

Answer: Distributed Version Control System (DVCS) for tracking changes in source code during software development.

Interview question
Core

Difference between Git and GitHub?

Answer: Git is the version control system; GitHub is a cloud-based hosting service for Git repositories.

Interview question
Core

What is a repository?

Answer: A directory that contains your project files and the entire history of changes (the .git folder).

Interview question
Core

What are the three states in Git?

Answer: Working directory (modified), staging area (staged), and repository (committed).

Interview question
Essential

What is .gitignore?

Answer: File listing patterns Git should ignore (build artifacts, dependencies, secrets). Prevents accidental commits.

Interview question
Core

What is a commit?

Answer: A snapshot of your repository at a specific point in time, identified by a unique SHA-1 hash.

Interview question
Core

What is HEAD?

Answer: A pointer to the current branch/commit you're working on. HEAD~1 means one commit before HEAD.

Interview question
Quick

What is the difference between git add . and git add -A?

Answer: git add . adds files in current directory; git add -A adds all changes in entire repository.

Interview question
Advanced

What is a detached HEAD state?

Answer: When HEAD points directly to a commit instead of a branch. Create a branch to avoid losing changes.

Interview question
Simple

How to check Git version?

Answer: git --version

Interview question

Topic 2: Branching & Merging

branch management (Google, Amazon, Microsoft, startups)

10 Questions
Core

What is a branch?

Answer: A movable pointer to a commit. Allows parallel development without affecting main codebase.

Interview question
Core

How to create and switch to a new branch?

Answer: git branch then git checkout , or git checkout -b (creates and switches).

Interview question
Advanced

Difference between git merge and git rebase?

Answer: Merge creates a merge commit with two parents; rebase rewrites commit objects with new parent.

Interview question
Core

What is a merge conflict?

Answer: When Git cannot automatically merge changes. You must manually resolve conflicting sections.

Interview question
Core

How to resolve a merge conflict?

Answer: Edit conflicted files, mark as resolved with git add, then complete merge with git commit.

Interview question
Theory

What is fast-forward merge?

Answer: When the branch being merged hasn't diverged. Git simply moves the pointer forward.

Interview question
Simple

How to delete a branch?

Answer: git branch -d (safe delete) or git branch -D (force delete).

Interview question
Quick

What is the difference between git switch and git checkout?

Answer: git switch is newer, branch-focused command; git checkout is older, multi-purpose command.

Interview question
Simple

How to rename a branch?

Answer: git branch -m or git branch -m (if currently on that branch).

Interview question
Theory

What is a tracking branch?

Answer: Local branch that tracks a remote branch. Automatically syncs with remote when using git pull/push.

Interview question

Topic 3: History & Logging

commit inspection (Google, Microsoft)

10 Questions
Core

How to view commit history?

Answer: git log (detailed), git log --oneline (compact), git log --graph (visual).

Interview question
Core

How to see changes in a specific commit?

Answer: git show or git show HEAD (for latest commit).

Interview question
Quick

What is git log --all --graph --oneline?

Answer: Shows all branches in a compact graph format, useful for visualizing branch structure.

Interview question
Maintenance

How to find who last modified a line?

Answer: git blame shows author and commit for each line.

Interview question
Quick

How to search commit messages?

Answer: git log --grep='' searches commit messages.

Interview question
Advanced

What is git reflog?

Answer: Reference log showing all HEAD movements. Useful for recovering lost commits.

Interview question
Core

How to see changes between commits?

Answer: git diff or git diff HEAD~1 HEAD (previous vs current).

Interview question
Quick

How to see commits in a date range?

Answer: git log --since='2024-01-01' --until='2024-12-31'.

Interview question
Advanced

What is git log --follow?

Answer: Follows file renames, showing history even after file was moved/renamed.

Interview question
Quick

How to see commits by author?

Answer: git log --author='' filters commits by author.

Interview question

Topic 4: Remotes & Collaboration

push, pull, fetch (Amazon, Microsoft, Google, startups)

10 Questions
Core

Difference between git fetch and git pull?

Answer: fetch downloads objects and refs but doesn't merge; pull = fetch + merge (or rebase with --rebase). Use fetch to inspect before merging.

Interview question
Workflow

How to set upstream branch?

Answer: git push -u origin feature or git branch --set-upstream-to=origin/feature.

Interview question
Quick

How to list remotes and show URLs?

Answer: git remote -v.

Interview question
Safe

How do you fetch from a remote and inspect changes before merging?

Answer: git fetch origin, then git log HEAD..origin/main --oneline or git diff HEAD..origin/main.

Interview question
Maintenance

How to handle a remote branch that someone deleted?

Answer: git fetch --prune removes stale remote-tracking refs; remove local branch if needed.

Interview question
Release

How to push a new tag to remote?

Answer: Create: git tag v1.0, then push: git push origin v1.0.

Interview question
Common

How to revert a pushed commit?

Answer: Prefer git revert which creates a new commit undoing changes (safe for shared history).

Interview question
OSS

How to sync your fork with upstream repo?

Answer: Add upstream remote: git remote add upstream , then git fetch upstream and git merge upstream/main (or rebase).

Interview question
Theory

What is remote-tracking branch?

Answer: Local refs like origin/main that track remote branch states; they're updated by git fetch/pull.

Interview question
Edge

How to clone only a single branch?

Answer: git clone --single-branch --branch (saves bandwidth).

Interview question

Topic 5: Stash & Working Directory

temporary saves (Microsoft, startups)

10 Questions
Core

What is git stash?

Answer: Temporarily saves uncommitted changes (working directory and index) so you can switch branches cleanly.

Interview question
Simple

How to stash with a message?

Answer: git stash push -m 'message'.

Interview question
Simple

How to list stashes?

Answer: git stash list shows all stashes with their messages.

Interview question
Core

How to apply a stash without removing it?

Answer: git stash apply (keeps stash) vs git stash pop (removes after applying).

Interview question
Advanced

How to stash only staged changes?

Answer: git stash push --staged (stashes only index, keeps working directory changes).

Interview question
Advanced

How to create a branch from a stash?

Answer: git stash branch creates branch and applies stash.

Interview question
Simple

How to drop a specific stash?

Answer: git stash drop stash@{0} removes the stash at index 0.

Interview question
Theory

What is the difference between git stash and git commit?

Answer: Stash is temporary and local; commit is permanent and can be pushed. Use stash for quick context switches.

Interview question
Quick

How to see what's in a stash?

Answer: git stash show -p stash@{0} shows the diff of stash contents.

Interview question
Simple

How to clear all stashes?

Answer: git stash clear removes all stashes permanently.

Interview question

Topic 6: Diff & Blame

code inspection (Google, Microsoft)

10 Questions
Core

What is git diff?

Answer: Shows changes between working directory, staging area, or commits.

Interview question
Core

Difference between git diff and git diff --staged?

Answer: git diff shows unstaged changes; git diff --staged shows staged changes.

Interview question
Quick

How to see diff of a specific file?

Answer: git diff or git diff HEAD (compares with last commit).

Interview question
Maintenance

What is git blame?

Answer: Shows who last modified each line of a file and in which commit.

Interview question
Quick

How to ignore whitespace in diff?

Answer: git diff -w or git diff --ignore-all-space ignores whitespace changes.

Interview question
Core

How to see diff between two branches?

Answer: git diff branch1..branch2 or git diff branch1 branch2.

Interview question
Quick

What is git diff --stat?

Answer: Shows summary statistics (files changed, insertions, deletions) instead of full diff.

Interview question
Core

How to see what changed in a file between commits?

Answer: git diff -- shows changes to specific file.

Interview question
Quick

What is git diff --cached?

Answer: Same as git diff --staged, shows changes in staging area.

Interview question
Advanced

How to see diff with context lines?

Answer: git diff -U sets number of context lines (default is 3).

Interview question

Topic 7: Undo & Reset

reverting changes (Amazon, Google, Microsoft)

10 Questions
Core

Difference between git reset and git revert?

Answer: reset moves HEAD pointer (rewrites history); revert creates new commit undoing changes (safe for shared history).

Interview question
Advanced

What is git reset --soft?

Answer: Moves HEAD but keeps changes staged. Useful for amending commit message.

Interview question
Dangerous

What is git reset --hard?

Answer: Moves HEAD and discards all changes. Dangerous - use with caution.

Interview question
Core

How to undo last commit but keep changes?

Answer: git reset --soft HEAD~1 (keeps changes staged) or git reset HEAD~1 (keeps changes unstaged).

Interview question
Modern

What is git restore?

Answer: Newer command to restore files from index or HEAD. Replaces git checkout for files.

Interview question
Core

How to unstage a file?

Answer: git restore --staged or git reset HEAD .

Interview question
Core

How to discard changes in working directory?

Answer: git restore or git checkout -- (older syntax).

Interview question
Theory

What is the difference between git reset and git restore?

Answer: reset moves branch pointer; restore only affects files in working directory/staging area.

Interview question
Advanced

How to undo a merge?

Answer: git reset --hard HEAD~1 (if not pushed) or git revert -m 1 (if pushed).

Interview question
Maintenance

What is git clean?

Answer: Removes untracked files. Use git clean -n (dry run) first, then git clean -f (force).

Interview question

Topic 8: Workflows & Best Practices

Gitflow, trunk-based (Netflix, Amazon, Google)

10 Questions
Model

What is Gitflow?

Answer: Branching model with main, develop, feature, release, hotfix branches. main is production, develop is integration.

Interview question
Model

What is trunk-based development?

Answer: Workflow where developers commit directly to main branch, using feature flags for incomplete features.

Interview question
Essential

What is a feature branch?

Answer: Branch created for developing a new feature, merged back when complete.

Interview question
Release

What is a release branch?

Answer: Branch for preparing a release, allows bug fixes without affecting develop.

Interview question
Release

What is a hotfix branch?

Answer: Branch for urgent production fixes, branched from main and merged to both main and develop.

Interview question
Release

What is semantic versioning?

Answer: Versioning scheme: MAJOR.MINOR.PATCH (e.g., 1.2.3). Used with git tags for releases.

Interview question
Essential

What is .gitignore and why use it?

Answer: File listing patterns Git should ignore (build artifacts, dependencies, secrets). Prevents accidental commits.

Interview question
Security

How to handle secrets in Git?

Answer: Never commit them. Use .gitignore, environment variables, or secret management tools. If committed, rotate secrets immediately.

Interview question
Advanced

What is the difference between merge and rebase at object level?

Answer: Merge creates a merge commit with two parents; rebase rewrites commit objects with new parent.

Interview question
Essential

Should you commit large binary files?

Answer: Generally no; use Git LFS or external storage. Git isn't optimized for binaries.

Interview question

Interview question

What is a Git hook?

Interview question

Interview question

What is git bisect?

Interview question
Advanced

What is git cherry-pick?

Interview question
Essential

What is git reflog?

Answer: Reference log showing all HEAD movements. Useful for recovering lost commits.

Interview question
Advanced

What is a submodule?

Answer: A Git repository inside another Git repository. Useful for including external projects.

Interview question
Maintenance

What is git gc?

Interview question
Maintenance

What is git fsck?

Interview question
Security

How to handle secrets in Git?

Answer: Never commit them. Use .gitignore, environment variables, or secret management tools. If committed, rotate secrets immediately.

Topic 10: GitHub/GitLab Features

PRs, protected branches, CI (GitHub, GitLab)

10 Questions
Core

What is a Pull Request (PR) / Merge Request (MR)?

Answer: Proposal to merge changes. Allows code review, discussion, and CI checks before merging.

Interview question
Integration

What is a protected branch?

Answer: Branch with restrictions (requires PR, approvals, CI checks) preventing direct pushes. Common for main/master.

Interview question
Integration

What is CI/CD in Git context?

Answer: Continuous Integration/Deployment - automated tests and deployments triggered by Git events (push, PR).

Interview question
OSS

What is a fork?

Answer: Personal copy of someone else's repository. Allows you to make changes without affecting original.

Interview question
Integration

What is a GitHub Action?

Answer: Automated workflow triggered by Git events. Used for CI/CD, testing, deployments.

Interview question
Essential

What is a code review?

Answer: Process where team members review code changes before merging. Improves code quality.

Interview question
Common

What is a merge conflict in a PR?

Answer: When PR branch and target branch have conflicting changes. Must resolve before merging.

Interview question
Common

What is squash and merge?

Answer: Merges all commits from PR into a single commit on target branch. Keeps history clean.

Interview question
Workflow

What is a draft PR?

Answer: Pull request marked as work-in-progress. Prevents accidental merging while still allowing review.

Interview question
Release

What is a GitHub release?

Topic 11: Troubleshooting & Recovery

fixing mistakes (all companies)

10 Questions
Essential

How to recover a deleted branch?

Answer: Use git reflog to find the commit, then git branch to recreate branch.

Interview question
Common

What to do if you committed to wrong branch?

Answer: Create new branch from current: git branch , reset original: git reset --hard origin/.

Interview question
Common

How to fix 'Your branch is ahead of origin'?

Answer: Push your commits: git push origin . Or reset if you want to discard: git reset --hard origin/.

Interview question
Common

What is 'detached HEAD' and how to fix?

Answer: HEAD points to commit instead of branch. Create branch: git branch or git checkout -b .

Interview question
Dangerous

How to undo a git push?

Answer: If not shared: git reset --hard HEAD~1 then git push --force. If shared: use git revert instead.

Interview question
Core

What to do if merge conflict occurs?

Answer: Edit conflicted files (look for <<<<<< markers), git add resolved files, then git commit to complete merge.

Interview question
Common

How to remove a file from Git but keep locally?

Answer: git rm --cached removes from index but keeps in working directory.

Interview question
Simple

What is 'fatal: not a git repository'?

Answer: You're not in a Git repository. Run git init or navigate to a Git repository directory.

Interview question
Common

How to change last commit message?

Answer: git commit --amend -m 'new message' (if not pushed) or create new commit with correct message.

Interview question
Common

What to do if 'git pull' fails?

Answer: Check for conflicts: git status. Resolve conflicts, then git add and git commit. Or use git pull --rebase.

Interview question

Topic 12: Configuration & Customization

settings, aliases, hooks (all companies)

10 Questions
Essential

How to set your name and email in Git?

Answer: git config --global user.name 'Your Name' and git config --global user.email 'your@email.com'.

Interview question
Core

What is git config --global?

Answer: Sets configuration for all repositories on your system. Use --local for repository-specific settings.

Interview question
Quick

How to create Git aliases?

Answer: git config --global alias.st status creates 'git st' as shortcut for 'git status'.

Interview question
Modern

How to set default branch name?

Answer: git config --global init.defaultBranch main sets 'main' as default instead of 'master'.

Interview question
Theory

What is .gitconfig file?

Answer: Configuration file storing Git settings (usually in ~/.gitconfig for global settings).

Interview question
Quick

How to see all Git configuration?

Answer: git config --list shows all settings, or git config --list --show-origin shows where each setting comes from.

Interview question
UI

How to set default editor for Git?

Answer: git config --global core.editor 'vim' (or 'code --wait' for VS Code, 'nano', etc.).

Interview question
Advanced

What is a Git hook?

Answer: Scripts that run automatically at certain Git events (pre-commit, post-merge, etc.).

Interview question
UI

How to enable Git autocomplete?

Answer: Source Git's bash completion script: source /usr/share/git-completion/git-completion.bash (location varies).

Interview question
Security

How to set up SSH keys for Git?

Answer: Generate: ssh-keygen -t ed25519, add to GitHub/GitLab: copy ~/.ssh/id_ed25519.pub to account settings.

Topic 13: Git Internals

objects, refs, hashing, packfiles (Google, Microsoft)

10 Questions
Advanced

What are the four types of Git objects?

Answer: Blobs (file content), trees (directories), commits (snapshots), and tags (annotated tags).

Interview question
Advanced

What is a SHA-1 hash in Git?

Answer: 40-character hexadecimal string uniquely identifying each object. Based on object content.

Interview question
Advanced

What is .git/objects directory?

Answer: Stores all Git objects (blobs, trees, commits). Objects are hashed and stored by first two hash characters.

Interview question
Advanced

What is .git/refs directory?

Answer: Stores references (branches, tags). refs/heads/ contains branch pointers, refs/tags/ contains tags.

Interview question
Advanced

What is a packfile?

Answer: Compressed collection of Git objects. Git automatically packs objects to save space.

Interview question
Advanced

What is the Git index?

Answer: Staging area - a binary file (.git/index) that stores information about files to be committed.

Interview question
Advanced

What is HEAD file?

Answer: File in .git/HEAD pointing to current branch or commit. Contains ref: refs/heads/main or commit hash.

Interview question
Advanced

What is git cat-file?

Answer: Command to inspect Git objects. git cat-file -p shows object content, -t shows object type.

Interview question
Advanced

What is a loose object vs packed object?

Answer: Loose: individual object file. Packed: compressed in packfile. Git automatically packs loose objects.

Interview question
Advanced

How does Git store file content?

Answer: As blobs (binary large objects) in .git/objects. Each unique file content gets one blob, referenced by multiple commits if unchanged.
