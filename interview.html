Topic 1 — Basics

What Git is / core commands (Amazon, Microsoft, Google, GitHub, Atlassian)

10 Questions
Fundamental

What is Git?

Answer: Distributed version control system that tracks snapshots of a project, supports branching/merging locally, and syncs via remotes.

Asked at: Amazon, Microsoft
Fundamental

What's the difference between Git and SVN?

Answer: Git is distributed (every clone has full history); SVN is centralized. Git generally faster for branching/merging and offline work.

Asked at: Google, Atlassian
Essential

What does git init do?

Answer: Creates a new local Git repository by making a .git directory storing objects, refs, and config.

Asked at: entry-level roles across companies
Essential

What does git clone do?

Answer: Copies a remote repository into a new local directory and sets origin remote.

Asked at: many companies
Essential

Explain git status.

Answer: Shows working tree state: staged, modified, untracked files and current branch.

Common interviewer starter
Essential

When do you use git add vs git commit?

Answer: git add stages changes to the index; git commit records staged changes to history.

Asked widely
Fundamental

What is a commit?

Answer: A snapshot with metadata (author, timestamp, message) referencing a tree of files.

Fundamental concept
Essential

Explain git log and useful flags.

Answer: Shows commit history. Flags: --oneline, --graph, --decorate, -p for diffs, --stat for file stats.

Asked at: engineering interviews
Useful

How to undo the last commit but keep changes?

Answer: git reset --soft HEAD~1 — undo commit, keep staged changes.

Asked by companies testing safe undo
Useful

How to discard local unstaged changes?

Answer: git checkout -- <file> (older) or git restore <file> (newer). To discard all: git reset --hard (be careful).

Asked at: many companies

Topic 2 — Branching & Merging

Everyday VCS skills (Google, Facebook/Meta, Microsoft, Netflix)

10 Questions
Common

What is a branch in Git?

Answer: A movable pointer to a commit (ref). Branches are lightweight and store the commit history pointer.

Common question
Practical

How do you create and checkout a branch?

Answer: git checkout -b feature or git switch -c feature.

Interview practical
Common

How to merge a branch into main?

Answer: git checkout main then git merge feature. If conflicts, resolve and git commit.

Common question
Important

What is a fast-forward merge?

Answer: When main has no new commits, merging moves main pointer to feature tip (no merge commit).

Often tested
Strategy

When would you prefer a merge commit vs squashing?

Answer: Keep merge commits to preserve feature branch history (big changes); squash for tidy single-commit feature history.

Behavioral/strategy question
Practical

How to resolve a merge conflict?

Answer: Edit conflicted files, remove conflict markers, test, then git add and git commit.

Practical question
Operational

How to delete a branch locally and remotely?

Answer: Local: git branch -d feature (or -D to force). Remote: git push origin --delete feature.

Operational question
Useful

What is git branch --merged?

Answer: Lists branches whose changes are already merged into current branch (useful for cleanup).

Interview tip
Edge-case

How do you rename a branch?

Answer: Local: git branch -m old new; push rename: git push origin :old new then set upstream.

Edge-case question
Advanced

How to check what commits a branch has ahead/behind another?

Answer: git log --oneline A..B shows commits in B not in A; git rev-list --left-right --count A...B gives counts.

Advanced basic

Topic 3 — Rebase, Cherry-pick, History Rewrites

Clean history (Atlassian, GitHub, Dropbox)

10 Questions
Critical

What does git rebase do?

Answer: Moves commits onto a new base (rewrites history) to create a linear history.

Critical concept
Practical

How to rebase a feature branch onto main?

Answer: git checkout feature then git rebase main. Resolve conflicts as needed, then git push --force-with-lease.

Safe practice
Important

When should you avoid rebasing?

Answer: Avoid rebasing public/shared branches (rewrites history causes problems for others). Use on local or private feature branches.

Best-practice
Useful

What is git cherry-pick?

Answer: Apply a specific commit from another branch onto the current branch: git cherry-pick <commit>.

Useful utility
Common

How to squash commits?

Answer: Interactive rebase: git rebase -i HEAD~N, mark s (squash) for commits to combine, edit message.

Common question
Simple

How to edit the last commit message?

Answer: git commit --amend -m "new message".

Simple
Practical

How to uncommit (move commit back to working directory)?

Answer: git reset --soft HEAD~1 (keeps changes staged) or --mixed to unstage.

Practical
Rescue

What is git reflog and when to use it?

Answer: A local log of HEAD updates; use to recover lost commits after reset or other history rewrites. (git reflog + git checkout can restore).

Often asked as rescue question
Advanced

How to remove a commit from middle of history?

Answer: Use git rebase -i and drop the commit, or create a branch from before the commit and cherry-pick desired commits.

Advanced
Best Practice

How to safely force-push after rewriting history?

Answer: git push --force-with-lease — safer than --force because it checks remote hasn't moved.

Best practice
