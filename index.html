Basic Commands

Essential Git commands to get you started with version control

git init Essential

Initializes a new Git repository in the current directory. This creates a hidden .git folder that contains all repository metadata, history, and configuration files.

Command 
git init
Example: Navigate to your project folder and run git init to start tracking changes.
git add Essential

Stages files for the next commit. You can add specific files, directories, or use . to add all changes. Staging allows you to selectively choose what goes into your commit.

Command 
git add .                    # Add all changes
git add file.txt            # Add specific file
git add src/                # Add entire directory
git add -A                  # Add all changes including deletions
Example: After modifying files, use git add . to stage all changes before committing.
git commit Essential

Creates a snapshot of your staged changes with a descriptive message. Each commit has a unique hash and represents a point in your project's history that you can return to.

Command 
git commit -m "Initial commit"
git commit -m "Add user authentication"
git commit -am "Quick commit"  # Add and commit tracked files
git commit --amend             # Modify the last commit
Example: git commit -m "Fix login bug" saves your staged changes with a clear message describing what was fixed.
git status Essential

Shows the current state of your working directory and staging area. It displays which files are modified, staged, or untracked, helping you understand what changes are ready to commit.

Command 
git status
git status -s                # Short format
git status --short           # Same as -s
Example: Run git status to see if you have uncommitted changes or files that need to be staged.
git diff Useful

Shows the differences between your working directory and the staging area, or between commits. Essential for reviewing changes before committing.

Command 
git diff                     # Unstaged changes
git diff --staged            # Staged changes
git diff HEAD                # All changes (staged + unstaged)
git diff commit1 commit2     # Compare two commits
Example: Use git diff to see exactly what lines were changed in your files before committing.

Branching

Create isolated environments for features, experiments, and bug fixes

git branch Essential

Lists all branches or creates a new branch. Branches allow you to work on features independently without affecting the main codebase.

Command 
git branch                    # List all branches
git branch feature-login     # Create new branch
git branch -d feature-login  # Delete merged branch
git branch -D feature-login  # Force delete branch
Example: Create a feature branch with git branch feature-login, then switch to it to start working on login functionality.
git switch Essential

Switches to another branch. This is the modern way to change branches (replaces git checkout for branch switching). Your working directory updates to match the branch you switch to.

Command 
git switch main              # Switch to main branch
git switch feature-login     # Switch to feature branch
git switch -c new-feature    # Create and switch to new branch
git switch -                 # Switch to previous branch
Example: Use git switch feature-login to move from main to your feature branch and start coding.

Merging

Combine changes from different branches into one unified history

git merge Essential

Combines changes from one branch into another. Creates a merge commit that ties together the histories of both branches. Essential for integrating feature work back into main.

Command 
git merge feature-login      # Fast-forward or merge commit
git merge --no-ff feature    # Always create merge commit
git merge --squash feature   # Squash all commits into one
git merge --abort            # Cancel merge in progress
Example: Switch to main with git switch main, then run git merge feature-login to integrate your feature.
Merge Conflicts Important

When Git can't automatically merge changes, you'll need to resolve conflicts manually. Git marks conflict areas in files, and you choose which changes to keep.

Command 
# After merge conflict appears:
git status                   # See conflicted files
# Edit files to resolve conflicts
git add .                    # Mark conflicts as resolved
git commit                   # Complete the merge
Example: If both branches modified the same line, Git will mark it with conflict markers. Edit the file to keep desired changes, then stage and commit.

History & Inspection

Explore your repository's commit history and understand what changed

git log Essential

Displays the commit history. With various flags, you can see a visual graph, filter by author, date, or search commit messages. Essential for understanding project evolution.

Command 
git log                      # Full history
git log --oneline            # Compact one-line format
git log --graph --decorate --all  # Visual branch graph
git log -5                   # Last 5 commits
git log --author="John"      # Filter by author
Example: git log --oneline --graph --all shows a beautiful visual representation of all branches and their relationships.
git show Useful

Shows detailed information about a specific commit, including the full diff of what changed. Perfect for reviewing what a commit did.

Command 
git show                     # Show latest commit
git show HEAD                # Show latest commit
git show abc1234             # Show specific commit
git show HEAD~1              # Show previous commit
Example: Use git show abc1234 to see exactly what files and lines were changed in that specific commit.

Undoing Work

Safely undo changes, commits, and mistakes with Git's powerful undo capabilities

git reset Use with Caution

Moves the branch pointer backward, effectively "undoing" commits. Different modes determine what happens to your working directory and staging area. Warning: Can lose work if not careful.

Command 
git reset --soft HEAD~1      # Undo commit, keep changes staged
git reset --mixed HEAD~1     # Undo commit, keep changes unstaged
git reset --hard HEAD~1      # Undo commit, discard all changes ‚ö†Ô∏è
Example: git reset --soft HEAD~1 undoes the last commit but keeps your changes staged, so you can fix the commit message and recommit.
git revert Safe

Creates a new commit that undoes the changes from a previous commit. This is the safe way to undo commits that have already been pushed to a shared repository, as it doesn't rewrite history.

Command 
git revert HEAD              # Revert last commit
git revert abc1234           # Revert specific commit
git revert --no-commit abc1234  # Revert without committing
Example: If you pushed a buggy commit, use git revert abc1234 to create a new commit that fixes it, without rewriting history.
git stash Essential

Temporarily saves your uncommitted changes so you can switch branches or pull updates with a clean working directory. Your changes are stored and can be reapplied later.

Command 
git stash                    # Save changes temporarily
git stash save "message"     # Stash with message
git stash list               # List all stashes
git stash pop                # Apply and remove latest stash
git stash apply              # Apply but keep stash
Example: Working on a feature but need to switch branches? Run git stash, switch branches, then git stash pop to restore your work.

Remote Repositories

Work with remote repositories like GitHub, GitLab, or Bitbucket

git remote Essential

Manages connections to remote repositories. Shows, adds, or removes remote URLs where you push and pull code from.

Command 
git remote -v                # List all remotes with URLs
git remote add origin URL    # Add a remote named 'origin'
git remote remove origin     # Remove a remote
git remote set-url origin NEW_URL  # Change remote URL
Example: After creating a GitHub repo, use git remote add origin https://github.com/user/repo.git to connect your local repo.
git push Essential

Uploads your local commits to a remote repository. This is how you share your work with others or back it up to services like GitHub.

Command 
git push origin main         # Push main branch to origin
git push origin feature      # Push feature branch
git push -u origin main      # Push and set upstream (first time)
git push --all               # Push all branches
Example: After committing locally, use git push origin main to upload your changes to GitHub for others to see.
git pull Essential

Fetches changes from remote and immediately merges them into your current branch. Equivalent to git fetch followed by git merge.

Command 
git pull                     # Fetch and merge from default remote
git pull origin main         # Pull from specific remote/branch
git pull --rebase            # Pull and rebase instead of merge
Example: git pull downloads the latest changes from GitHub and merges them into your current branch, keeping you up to date.
git clone Essential

Creates a copy of a remote repository on your local machine. This is how you get started with an existing project from GitHub or other Git hosting services.

Command 
git clone URL                # Clone repository
git clone URL folder-name    # Clone into specific folder
git clone --depth 1 URL      # Shallow clone (faster, less history)
Example: git clone https://github.com/user/repo.git downloads the entire project and sets up the remote connection automatically.

Repository Maintenance

Keep your repository clean, optimized, and healthy

git gc Maintenance

Garbage collection - optimizes your repository by compressing files and removing unreachable objects. Helps reduce repository size and improve performance.

Command 
git gc                       # Standard garbage collection
git gc --aggressive          # More thorough optimization
git gc --prune=now           # Remove unreachable objects immediately
Example: Run git gc --aggressive periodically on large repositories to optimize storage and improve Git performance.

Advanced Git

Powerful features for experienced users and complex workflows

git rebase Advanced

Rewrites commit history by replaying commits from one branch onto another. Creates a linear history but rewrites commits, so use carefully on shared branches.

Command 
git rebase main              # Rebase current branch onto main
git rebase -i HEAD~3         # Interactive rebase (last 3 commits)
git rebase --abort           # Cancel rebase in progress
git rebase --continue        # Continue after resolving conflicts
Example: git rebase main takes your feature branch commits and replays them on top of the latest main, creating a cleaner linear history.
git cherry-pick Advanced

Applies a specific commit from one branch to another. Useful for porting bug fixes or features between branches without merging entire branches.

Command 
git cherry-pick abc1234      # Apply specific commit
git cherry-pick -x abc1234   # Include original commit hash
git cherry-pick main..feature # Pick range of commits
Example: Found a bug fix in the main branch? Use git cherry-pick abc1234 to apply just that commit to your feature branch.
